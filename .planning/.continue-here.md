## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| `os.environ.get(key, default)` não cobre env var vazia | O GitHub Actions expõe `vars.X` não configurada como **string vazia**, não como chave ausente — `.get(key, default)` só cai no default quando a chave nem existe. Isso deixou `APIFY_ACTOR_INSTAGRAM` silenciosamente vazio, derrubando 100% dos links de Instagram por dias sem erro óbvio. | blocking | Corrigido em `apify_fetch.py`: `os.environ.get(key) or default`. Para qualquer env var opcional nova, usar sempre o padrão `or`, nunca o segundo argumento do `.get()`. |
| Template do código contradizendo o sistema viral | O `FORMATO_SAIDA` antigo prescrevia bloco isolado `[25-50s - Explicação do Problema]` e CTA genérica nas 3 versões — o oposto do que o SISTEMA_VIRAL (Seções 7 e 9) manda (estrutura intercalada; CTA RECEITA só na shorts). Resultado: roteiros violando o próprio sistema (queixa direta do Luiz). | blocking | Corrigido em 2026-07-19 (commit `f19d744`): template reescrito como estrutura intercalada (4 frases por ingrediente, explicação diluída) + hierarquia de CTA fixa, com checagens programáticas em `_avisos_qa()`. Se mexer no `FORMATO_SAIDA` de novo, validar contra as Seções 7 e 9 do SISTEMA_VIRAL antes — nunca reintroduzir bloco isolado de explicação nem CTA de comentário na mãe/rápida. |
| Retry de tamanho dobrava o custo | O retry re-enviava o system prompt inteiro (na época ~78k tokens de KB) e disparava quase sempre → ~$0,31/vídeo. | blocking | Retry REMOVIDO por decisão explícita do Luiz ("método definitivo, é aquilo ou nada"). Tamanho garantido por contrato estrutural + overshoot no prompt (mire 3.800-4.000, mínimo 3.600). Desvios viram avisos de QA (log + callout vermelho no Notion + Telegram), nunca segunda chamada. NÃO reintroduzir retry. |
| Cron do GitHub Actions não é confiável pra tempo real | Gaps reais de 3h15min e 5h+ entre execuções agendadas confirmados via API. | advisory | Detecção de mensagem migrada pro Cloudflare Worker (webhook). `queue_flush.py` é só rede de segurança (cron 30 min). Se algo "não disparar", checar logs do Worker primeiro. |
| GitHub API exige header `User-Agent` | Chamada sem `User-Agent` volta 403 "Request forbidden by administrative rules". | advisory | `githubRequest()` em `worker.js` sempre envia o header. Chamadas novas fora dela precisam lembrar disso (inclusive scripts locais via curl/urllib). |

**Severity values:** `blocking` — resuming agent must pass understanding check before proceeding. `advisory` — important context, does not gate resumption.
