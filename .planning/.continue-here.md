## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Conflito de instruções de formato no prompt | A base de conhecimento (KB) enviada como parte do system prompt do `generate_script.py` tem sua própria seção "FORMATO DE ENTREGA OBRIGATÓRIO" (pensada pro uso manual do agente Cowork, com emojis/blocos de análise) que competia com o formato de tags simples que o `roteiro_parser.py` espera. O Claude seguia o formato da KB, e o parser não reconhecia nada — resultado: card criado no Notion com todos os campos vazios (título genérico "Roteiro minerado"), sem lançar erro. | blocking | Já corrigido: `generate_script.py` agora tem uma instrução explícita reforçando que a seção 11 da KB deve ser ignorada, e `notion_insert.py` agora lança `RuntimeError` se o parsing vier vazio (título E versão-mãe ambos vazios), em vez de criar o card silenciosamente. Se você (ou um agente futuro) for mexer no prompt de `generate_script.py` de novo, cuidado pra não reintroduzir esse conflito — releia o comentário adicionado antes do bloco de instrução. |
| Suposição desatualizada sobre o runner do GitHub Actions | O comentário original em `transcribe.py` dizia "ubuntu-latest já vem com ffmpeg instalado" — isso não é mais verdade. | advisory | Corrigido: passo `apt-get install -y ffmpeg` adicionado no `process-video.yml`. Se outro workflow novo precisar de ffmpeg, lembre de adicionar o mesmo passo — não assumir que já vem instalado. |
